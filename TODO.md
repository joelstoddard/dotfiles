### Bugs discovered during drift audit

- [x] **Top-level `check:` fields in `packages.yaml` aren't resolved.** `lib/packages.py::_resolve_spec` only reads `check` from the per-OS dict, so `nvm` (which has `check: "test -d $HOME/.nvm"` at the package level) still reports `[dry-run] would install` even though the check would pass. Same likely affects `oh-my-posh`, `claude-code`, `ttl`, `rust`, `aws-cli`, `docker`, `golang`, `colima`, `libpq`, `python`, `vscode`, `granted`, `jfrog`, `bitwarden-cli`, `ibm-plex-mono`, `atkinson-hyperlegible`, `bitwarden` — all of which have top-level `check:`. Fix: have `resolve()` pull `check` from `pkg_data` and merge into the PackageSpec alongside the per-OS attrs.
